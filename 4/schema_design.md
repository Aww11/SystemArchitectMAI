## 1. Выбор между embedded и references

### 1.1 Связь "пользователь — корзина"

**Решение:** Embedded (корзина встроена в документ пользователя)

**Структура:**
```json
{
  "_id": ObjectId,
  "username": "ivan_88",
  "email": "ivan@mail.ru",
  "first_name": "Иван",
  "last_name": "Петров",
  "cart": [
    {
      "product_id": "000000000000000000000001",
      "product_name": "iPhone 15 Pro",
      "quantity": 1,
      "price": 999.99,
      "added_at": ISODate("2024-02-01T10:00:00Z")
    }
  ]
}
```

**Обоснование:**
- Корзина всегда принадлежит одному пользователю
- Данные корзины нужны вместе с пользователем при каждом запросе
- В задании нет отдельной сущности "корзина"
- Количество товаров в корзине небольшое (1-10), лимит 16MB не превышается
- При оформлении заказа корзина очищается одной операцией

### 1.2 Связь "пользователь — товар" (через корзину)

**Решение:** References (ссылка на товар по ID)

**Обоснование:**
- Товары общие для всех пользователей
- Изменение цены или названия товара должно отражаться во всех корзинах
- При embedded пришлось бы обновлять каждого пользователя
- Поля `product_name` и `price` дублируются в корзине ради скорости чтения

### 1.3 Коллекция products

**Решение:** Отдельная коллекция

**Обоснование:**
- Товары не зависят от пользователей
- Поиск, фильтрация и сортировка товаров — частые операции
- Отдельная коллекция позволяет эффективно индексировать поля

## 2. Итоговая схема коллекций

### Коллекция `users`

| Поле | Тип | Описание |
|------|-----|----------|
| _id | ObjectId | Уникальный идентификатор |
| username | string | Логин (уникальный) |
| email | string | Email (уникальный) |
| first_name | string | Имя |
| last_name | string | Фамилия |
| hashed_password | string | Хеш пароля |
| created_at | date | Дата регистрации |
| is_active | boolean | Активен ли пользователь |
| cart | array | Массив товаров в корзине |

**Cart item (вложенный документ):**

| Поле | Тип | Описание |
|------|-----|----------|
| product_id | string | ID товара (ссылка на products) |
| product_name | string | Название товара (дублируется) |
| quantity | int | Количество |
| price | float | Цена на момент добавления |
| added_at | date | Дата добавления |

### Коллекция `products`

| Поле | Тип | Описание |
|------|-----|----------|
| _id | ObjectId | Уникальный идентификатор |
| name | string | Название товара |
| description | string | Описание |
| price | float | Цена |
| stock | int | Остаток на складе |
| category | string | Категория |
| created_at | date | Дата добавления |

---

## 3. Схема связей

```
┌─────────────┐         ┌─────────────┐
│    users    │         │  products   │
├─────────────┤         ├─────────────┤
│ _id         │         │ _id         │
│ username    │         │ name        │
│ email       │         │ price       │
│ first_name  │         │ stock       │
│ last_name   │         │ category    │
│ cart ───────┼────────►│ ...         │
└─────────────┘  ссылка  └─────────────┘
   (embedded)    по _id      (отдельная)
```

- Связь "многие-ко-многим" между пользователями и товарами
- Реализована через массив `cart` в коллекции `users`
- Каждый элемент массива содержит `product_id` (ссылку на `products._id`)

---

## 4. Сравнение подходов

| Критерий | Embedded в users | Отдельная коллекция cart |
|----------|------------------|--------------------------|
| Запросов для получения корзины | 1 | 2 |
| Обновление корзины | updateOne | deleteMany + insertMany |
| Дублирование данных | Да (название, цена) | Нет |
| Размер документа | Ограничен 16MB | Неограничен |
| Скорость чтения | Высокая | Ниже |

**Вывод:** Для интернет-магазина с небольшим количеством товаров в корзине embedded подход даёт лучшую производительность.

## 5. Индексы

| Коллекция | Индекс | Назначение |
|-----------|--------|------------|
| users | username (уникальный) | Поиск по логину |
| users | email (уникальный) | Поиск по email |
| users | first_name | Поиск по маске имени |
| users | last_name | Поиск по маске фамилии |
| products | category | Фильтрация по категории |
| products | name | Поиск по названию |
| products | price | Сортировка по цене |

MongoDB автоматически создаёт индекс по полю `_id` для каждой коллекции.
